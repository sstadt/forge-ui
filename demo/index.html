Transitions

To use any of the transitions, simple add the keyword to the transition name to the element you will be showing/hiding as described in the Vue documentation.

The following transitions are available as part of MyUI:

  • fade
  • slide-up
  • slide-down
  • zoom-in
  • zoom-out

Alert

The alert component is designed to have it's methods accessed via ref in a parent component. Calling the appropriate method attached to the ref will render an error in the box. Setting the error as a new type will clear the previous errors in the alert.

Params

canClose
Boolean; defaults to false
Set to true to display the close button in the top right corner.

Methods

message
Display a callout message with no additional color class.
primary
Display a callout message with the primary color class.
secondary
Display a callout message with the secondary color class.
success
Display a callout message with the success color class.
warning
Display a callout message with the warning color class.
alert
Display a callout message with the alert color class.
 

<alert v-ref:my-alert :can-close="true"></alert>
<div class="row">
  <div class="small-12 medium-5 columns">
    <v-input type="text" label="Alert Message" :value.sync="alertMessage"></v-input>
  </div>
  <div class="small-12 medium-5 columns">
    <v-select label="Method" :selected.sync="selectedMessageType" :options="messageTypes" :required="true"></v-select>
  </div>
  <div class="small-12 medium-2 columns">
    <button type="button" class="button expanded inline-form" v-on:click="newMessage()">Send</button>
  </div>
</div>


new Vue({
  el: '#alert_demo',
  data: {
    alertMessage: '',
    selectedMessageType: 'message',
    messageTypes: [
      { value: 'message', label: 'vm.$refs.myAlert.message()' },
      { value: 'primary', label: 'vm.$refs.myAlert.primary()' },
      { value: 'secondary', label: 'vm.$refs.myAlert.secondary()' },
      { value: 'success', label: 'vm.$refs.myAlert.success()' },
      { value: 'warning', label: 'vm.$refs.myAlert.warning()' },
      { value: 'alert', label: 'vm.$refs.myAlert.alert()' },
    ]
  },
  methods: {
    newMessage() {
      this.$refs.myAlert[this.selectedMessageType](this.alertMessage);
    }
  }
});

Modal

A simple modal dialog that is controlled by a single visibility variable

Params

show
Boolean; two-way; required
Controls modal visibility; true shows the modal, false hides it.
modalSize
String
Sets the size of the modal. Can be set to 'small' or 'full' to increase or decrease the size of the modal.

Prompt

A simple prompt component meant to replace the native prompt and confirm methods. This will allow you to style the dialog to match your site and prevent users from asking the browser to suppress further dialogs.

This component is meant to get input back from the user. If you simply need to notify the user of something, use an alert component to display the information and it's canClose property if the information should be dismissable.

With great power comes great responsibility. These dialogs cannot be suppressed by the browser, so take care not to inundate the user with an endless stream of repeditive dialogs.

Methods

ask
This provides a text input to the user to enter a custom answer similar to the native prompt() method. When clicking the confirm button the dialog input will be validated before calling the provided yes() function.
confirm
This provides a yes or no dialog similar to the native confirm() method.

Both prompt methods take an options object supporting the following parameters:

  • question the dialog label to present to the user
  • yesLabel the confirm button label
  • noLabel the cancel button label
  • yes a function to execute when the user clicks the confirm button
  • no a function to execute when the user clicks the cancel button

<prompt v-ref:my-prompt></prompt>
<button type="button" class="button" v-on:click="prompt('ask')">vm.$refs.myPrompt.ask()</button>
<button type="button" class="button" v-on:click="prompt('confirm')">vm.$refs.myPrompt.confirm()</button>


new Vue({
  el: '#prompt_demo',
  methods: {
    prompt(method) {
      this.$refs.myPrompt[method]({
        question: 'Judge me by my size, do you?',
        yes: function (answer) {
          // if method is 'ask',
          // the dialog input value will be passed as the first argument
        },
        no: function () {
          // the prompt will close itself,
          // but you can use this option to trigger side effects
        }
      });
    }
  }
});

vForm

The vForm component automatically validates it's child components on form submission and can be configured use default submission behavior or execute a callback function is inline or AJAX behavior is more desirable.

If there are any errors in the configured validation errors will be triggered on inputs and the form will not submit or execute the submitCallback.

Params

method
String; defaults to POST
Form submission method; supports GET and POST just like a standard form
action
String; defaults to an empty string
The URL to submit the form to, if not configured as async
async
Boolean; defaults to false
Set to true to override default form submission behavior; useful if you only want to trigger the submitCallback
submitCallback
Function
Callback function to execute on form submission

<alert v-ref:my-alert :can-close="true"></alert>
<v-form :async="true" :submit-callback="showMessage">
  <v-input type="text" :value.sync="message" :required="true"></v-input>
  <button type="submit" class="button">Submit</button>
</v-form>


new Vue({
  el: '#form_demo',
  data: {
    message: ''
  },
  methods: {
    showMessage() {
      this.$refs.myAlert.success(this.message);
    }
  }
});

vInput

The vInput component provides a quick and easy way to add a standardized form input with automatic validation. The vInput directive can generate a textarea or any html5 input type. Validation occurs automatically on blur.

Params

type
String; required
The input type; set to textarea to render a textarea instead of an input
value
String; required; two-way
The input value
label
String
The input label text
name
String
The name attribute for the input
placeholder
String
The input placeholder text
required
Boolean; defaults to false
True if the input should be required; see forms
equalTo
String; defaults to null
Set to another observable to test equivalency

<v-input type="email" label="Email" :value.sync="email" />


new Vue({
  el: '#input_demo',
  data: {
    email: ''
  }
});

vSelect

Renders a select input given a set of options

Params

label
String; defaults to an empty string
The label to render for the select input
selected
String; required; two-way
The selected value
options
Array; required
An array of options to render in the select menu; each array element should be an object with a label and value parameter
required
Boolean; defaults to false
When set to false, will render a select option with no value

selected: {{ selectedOption }}

 

<v-select :selected.sync="selectedOption" :options="selectOptions"></v-select>


new Vue({
  el: '#select_demo',
  data: {
    selectedOption: '',
    selectOptions: [
      { label: 'One Option', value: '1' },
      { label: 'Two Option', value: '2' },
      { label: 'Red Option', value: 'Red' },
      { label: 'Blue Option', value: 'Blue' }
    ]
  }
});
